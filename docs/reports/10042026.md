## Le Problème : Le Code "Trop Permissif" 🚨

Imagine que tu as une boîte où tu ranges tes affaires d'école. Si tu écris juste "affaires" sur l'étiquette, tu pourrais mettre n'importe quoi dedans : un livre, une pomme, ton téléphone... mais aussi des choses qui n'ont rien à faire là, comme du sable ou de l'eau !

En programmation, c'est pareil avec le mot `any` en TypeScript. C'est une étiquette qui dit "peut contenir n'importe quoi". C'est **dangereux** parce que :

1. **On ne sait pas ce qu'il y a vraiment** dans la variable
2. **Des erreurs peuvent se cacher** et apparaître seulement quand l'application plante
3. **Pour un système de santé**, c'est critique car on manipule des données de patients

Ton projet avait **35+ avertissements** de ce type, répartis dans plus de **20 fichiers** différents !

## Ce que j'ai fait (en simplifié) 🔧

### 1. J'ai remplacé les `any` par des types précis

**Avant :**
```typescript
async createPatient(@Body() fhirPayload: any) {
  // Peut recevoir n'importe quoi : un nombre, un texte, un objet bizarre...
}
```

**Après :**
```typescript
async createPatient(@Body() fhirPayload: unknown): Promise<FhirPatient> {
  // Doit recevoir un format FHIR spécifique
  // Et promet de retourner un patient FHIR bien formaté
}
```

**Analogies :**
- `any` = 🎁 Boîte mystère (peut contenir n'importe quoi)
- `unknown` = 📦 Boîte scellée (on doit vérifier avant d'ouvrir)
- `FhirPatient` = 🏥 Dossier patient structuré (format médical standard)

### 2. J'ai ajouté des "promesses de retour" explicites

Quand une fonction dit qu'elle va faire quelque chose, elle doit tenir sa promesse ! J'ai ajouté ce que chaque fonction **promet de retourner**.

**Exemple :**
```typescript
// Avant : "Je vais faire quelque chose, mais je ne te dis pas quoi"
async getPatient(id: string) { ... }

// Après : "Je te promets de te donner un patient FHIR"
async getPatient(id: string): Promise<FhirPatient> { ... }
```

### 3. J'ai créé des "passeports" pour les requêtes

Dans ton système, quand quelqu'un se connecte, l'application vérifie son identité (comme un passeport). J'ai créé un type spécial pour ça :

```typescript
type AuthenticatedFastifyRequest = FastifyRequest & { 
  user?: { userId: string } 
};
```

**Avant :**
```typescript
const user = (req as any).user; // ❌ "Fais-moi confiance, c'est un user"
```

**Après :**
```typescript
const user = req.user; // ✅ TypeScript sait que c'est bien un user
```

## Résultats 🏆

1. **Moins de bugs** - TypeScript peut maintenant détecter plus d'erreurs avant l'exécution
2. **Code plus lisible** - On sait ce que chaque fonction accepte et retourne
3. **Meilleure maintenance** - Plus facile pour toi ou d'autres de comprendre le code plus tard
4. **Sécurité améliorée** - Moins de risques d'erreurs avec les données sensibles de santé
