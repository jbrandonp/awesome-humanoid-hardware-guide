# 04.2 — Ajouter vérification JWT_SECRET au démarrage

**Réf** : `260426.md:1061-1063`  
**Priorité** : 🟠 P4 — SÉCURITÉ  
**Durée estimée** : 5 min  
**Dépend de** : P1

---

## Contexte

Si `JWT_SECRET` n'est pas configuré (reste à la valeur par défaut), le serveur démarre sans erreur et des JWT forgés peuvent être acceptés. Il faut bloquer le démarrage en cas de secret non configuré.

---

## Fichier concerné

```
apps/api/src/app/app.module.ts
```

---

## Action

Ajouter dans `main.ts` ou au début du bootstrap :

```ts
if (process.env.JWT_SECRET === 'production-secret-key-change-me' || !process.env.JWT_SECRET) {
  throw new Error('JWT_SECRET non configuré — démarrage bloqué');
}
```

---

## Vérification

```bash
# Sans JWT_SECRET défini → doit refuser de démarrer
unset JWT_SECRET
npx nx serve api
# Doit throw: JWT_SECRET non configuré
```

---

## Risques

- **Faible** : Protection standard
- En développement, utiliser une valeur par défaut différente ou désactiver le check
