# 04.6 — Ajouter @UseGuards sur iot.controller.ts

**Réf** : `260426.md:427` (IOT2) + `260426.md:1073`  
**Priorité** : 🟠 P4 — CRITIQUE SÉCURITÉ  
**Durée estimée** : 5 min  
**Dépend de** : P1

---

## Contexte

Les endpoints IoT (`/iot/ble/blood-pressure`, `/iot/smart-pen/ink`, `/iot/device/heartbeat`) sont sans `@UseGuards()`. N'importe qui peut injecter des données médicales.

---

## Fichier concerné

```
apps/api/src/app/iot/iot.controller.ts
```

---

## Action

Ajouter sur la classe :

```ts
@Controller('iot')
@UseGuards(AuthGuard('jwt'))  // ← ajouter
export class IotController {
  // ...
}
```

---

## Vérification

```bash
curl -X POST http://localhost:3000/api/iot/ble/blood-pressure -d '{}'
# Doit retourner 401

curl -X POST http://localhost:3000/api/iot/ble/blood-pressure \
  -H "Authorization: Bearer <token>" -d '{...}'
# Doit retourner 200
```

---

## Risques

- **Moyen** : Les dispositifs IoT doivent être configurés pour envoyer un token JWT
- Alternative : utiliser une API key distincte pour les dispositifs IoT (plus léger qu'un JWT)
