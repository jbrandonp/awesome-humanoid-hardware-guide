## Contexte

Le login accepte **n'importe quel mot de passe** et connecte toujours en tant que DOCTOR avec un ID fixe. Il faut implémenter une vraie vérification email/password avec bcrypt.

## Action

**Avant** (login hardcodé) :
```ts
@Post('login')
async login(@Body() body: LoginDto) {
  // N'importe quel mot de passe connecte
  return { userId: '123e4567-e89b-12d3-a456-426614174000', role: 'DOCTOR' };
}
```

**Après** (login réel) :
```ts
@Post('login')
async login(@Body() body: LoginDto) {
  const user = await this.prisma.user.findUnique({
    where: { email: body.email },
  });
  if (!user) throw new UnauthorizedException('Identifiants invalides');
  
  const valid = await bcrypt.compare(body.password, user.passwordHash);
  if (!valid) throw new UnauthorizedException('Identifiants invalides');
  
  const token = this.jwtService.sign({ sub: user.id, role: user.role });
  return { accessToken: token, userId: user.id, role: user.role };
}
```
