# 01.4 — Downgrade Jest 30 → 29

**Réf** : `260426.md:289` (C5) + `260426.md:791-805` (analyse détaillée)  
**Priorité** : 🔴 P1 — BLOQUANT BUILD  
**Durée estimée** : 5 min  
**Dépend de** : P0

---

## Contexte

`ts-jest@29` est incompatible avec `jest@30`. Comme `ts-jest@30` n'existe pas en version stable (beta uniquement), il faut downgrade tout l'écosystème Jest à v29.

---

## Fichier concerné

```
package.json (racine)
```

---

## Action

Modifier les versions suivantes dans `devDependencies` :

| Package | Avant | Après |
|---------|-------|-------|
| `jest` | `^30.0.2` | `^29.7.0` |
| `@types/jest` | `^30.0.0` | `^29.5.14` |
| `jest-environment-jsdom` | `^30.3.0` | `^29.7.0` |
| `jest-environment-node` | `^30.0.2` | `^29.7.0` |
| `jest-util` | `^30.0.2` | `^29.7.0` |

Garder `ts-jest: ^29.4.6` inchangé.

---

## Vérification

```bash
npx jest --version
# Doit afficher 29.x.x
```

---

## Risques

- **Moyen** : Des tests utilisant des APIs Jest 30 pourraient casser
- Si des breaking changes Jest 30 sont utilisés → identifier et adapter
- `@types/jest` doit matcher la version exacte de `jest`
