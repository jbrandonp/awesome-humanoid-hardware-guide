# 01.12 — Corriger waiting-room.module.ts : remplacer JwtModule vide par AuthModule

**Réf** : `260426.md:613` (AB14) + `260426.md:818-824` + `260426.md:966-969`  
**Priorité** : 🔴 P1 — BLOQUANT BUILD (JWT crash)  
**Durée estimée** : 3 min  
**Dépend de** : P0

---

## Contexte

`WaitingRoomModule` utilise `JwtModule.register({})` avec une config **vide**. Au `verify()`, crash car pas de secret. `AuthModule` exporte déjà un `JwtModule` correctement configuré avec `JWT_SECRET`.

---

## Fichier concerné

```
apps/api/src/app/waiting-room/waiting-room.module.ts:6
```

---

## Action

**Avant** :
```ts
imports: [JwtModule.register({}), /* ... */],
```

**Après** :
```ts
imports: [AuthModule, /* ... */],
```

```bash
npx nx typecheck api
```

## Risques

- **Faible** : AuthModule exporte déjà JwtModule configuré
- Vérifier qu'AuthModule n'a pas d'imports circulaires avec WaitingRoomModule
