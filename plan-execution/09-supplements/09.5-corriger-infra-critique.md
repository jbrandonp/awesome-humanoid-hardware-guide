# 09.5 — Corriger les bugs d'infrastructure critiques

## Contexte

4 bugs d'infrastructure critiques :
1. **INF2** : `docker-compose.prod.yml` healthcheck utilise `curl` absent des images Alpine → healthcheck toujours FAIL
2. **INF5** : `generate-local-certs.js` écrit les clés privées world-readable (`chmod 644` au lieu de `600`)
3. **INF7** : `init.sh` utilise `prisma db push` (prototypage) au lieu de `migrate deploy` → risque perte de données
4. **INF6** : `docker-compose.prod.yml` JWT secrets sans valeur par défaut → chaînes vides si env non défini

```
infra/docker/docker-compose.prod.yml

```

## Action

### 1. INF2 — Healthcheck sans curl

**Avant** :
```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
```

**Après** (utiliser `wget` ou `node -e`) :
```yaml
healthcheck:
  test: ["CMD", "node", "-e", "require('http').get('http://localhost:3000/health', (r) => process.exit(r.statusCode === 200 ? 0 : 1))"]
```

### 2. INF5 — Clés privées world-readable

**Avant** (`generate-local-certs.js:139`):
```js
fs.writeFileSync(keyPath, privateKey);
```

**Après** :
```js
fs.writeFileSync(keyPath, privateKey, { mode: 0o600 });
```

### 3. INF7 — prisma db push → migrate deploy

**Avant** (`init.sh:39`):
```sh
npx prisma db push
```

**Après** :
```sh
npx prisma migrate deploy
```

### 4. INF6 — JWT secrets par défaut

Ajouter des valeurs par défaut explicites dans `.env.example` ou docker-compose :
```yaml
environment:
  - JWT_SECRET=${JWT_SECRET:-change-me-in-production}
  - JWT_REFRESH_SECRET=${JWT_REFRESH_SECRET:-change-me-in-production}
```

## Risques

- **Moyen** : `migrate deploy` nécessite que le dossier `prisma/migrations/` soit à jour
- `db push` peut avoir créé un état de DB non synchronisé avec les migrations
