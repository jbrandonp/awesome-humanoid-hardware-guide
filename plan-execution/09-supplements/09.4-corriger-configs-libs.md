# 09.4 — Corriger les configs libs (jest, tsconfig)

**Réf** : `260426.md:298` (C14) + `260426.md:299` (C15) + `260426.md:306` (C17)  
**Priorité** : 🟠 MANQUANT — BLOQUANT TESTS LIBRAIRIES  
**Durée estimée** : 10 min  
**Dépend de** : P1

---

## Contexte

3 bugs de config dans les librairies :
1. **C14** : `cornerstone-integration/jest.config.js` référence un `tsconfig.json` racine qui a `"files": []` → **aucun fichier compilé pour les tests**
2. **C15** : `insurance-engine/tsconfig.lib.json` exclut `jest.config.ts` et `jest.config.cts` mais le vrai fichier s'appelle `jest.config.cjs` → **config Jest compilée dans le build**
3. **C17** : `cornerstone-integration/tsconfig.lib.json` a `"lib": ["es2015"]` → trop vieux, pas de `Promise.finally`, `Array.flat`

---

## Fichiers concernés

```
libs/cornerstone-integration/jest.config.js
libs/insurance-engine/tsconfig.lib.json
libs/cornerstone-integration/tsconfig.lib.json
```

---

## Action

### 1. `cornerstone-integration/jest.config.js`

**Avant** (référence cassée) :
```js
module.exports = {
  // ...
  transform: {
    '^.+\\.tsx?$': ['ts-jest', { tsconfig: 'tsconfig.json' }],
  },
};
```

**Après** :
```js
module.exports = {
  // ...
  transform: {
    '^.+\\.tsx?$': ['ts-jest', { tsconfig: 'tsconfig.spec.json' }],
  },
};
```

### 2. `insurance-engine/tsconfig.lib.json`

Ajouter `jest.config.cjs` aux exclusions :
```json
"exclude": ["jest.config.ts", "jest.config.cts", "jest.config.cjs", "**/*.spec.ts"]
```

### 3. `cornerstone-integration/tsconfig.lib.json`

Mettre à jour `lib` :
```json
"lib": ["es2022"]
```

---

## Vérification

```bash
npx nx test cornerstone-integration
npx nx build insurance-engine
```

## Risques

- **Faible** : Corrections de config
- Si `es2022` cause des problèmes → utiliser `es2021` (minimum pour `Array.flat`)
