# 09.7 — Corriger les conflits de config frontend et mobile

**Réf** : `260426.md:316` (C27) + `260426.md:324` (C30) + `260426.md:607` (AB7)  
**Priorité** : 🟡 MANQUANT — BUILD + SÉCURITÉ  
**Durée estimée** : 10 min  
**Dépend de** : P1

---

## Contexte

3 bugs de config :
1. **C27** : `apps/kiosk/vite.config.mts` et `apps/desktop/vite.config.mts` utilisent le **même port 4200** → conflit si lancés ensemble
2. **C30** : `expo-file-system: ^55.0.11` dans `apps/mobile` vs `^17.0.1` à la racine → conflit majeur de version
3. **AB7** : Webhook Meta dans `engagement.controller.ts` sans vérification `X-Hub-Signature-256`

---

## Fichiers concernés

```
apps/kiosk/vite.config.mts
apps/desktop/vite.config.mts
apps/mobile/package.json
apps/api/src/app/engagement/engagement.controller.ts
```

---

## Action

### 1. C27 — Changer le port kiosk

**`apps/kiosk/vite.config.mts`** :
```ts
server: { port: 4201 }  // au lieu de 4200
```

### 2. C30 — Aligner expo-file-system

**`apps/mobile/package.json`** :
```json
"expo-file-system": "^17.0.1"  // aligné avec la racine
```

### 3. AB7 — Ajouter vérification signature Meta

**`engagement.controller.ts`** :
```ts
@Post('webhook/meta')
async metaWebhook(
  @Headers('x-hub-signature-256') signature: string,
  @Body() body: any,
) {
  const expected = crypto
    .createHmac('sha256', process.env.META_APP_SECRET)
    .update(JSON.stringify(body))
    .digest('hex');

  if (`sha256=${expected}` !== signature) {
    throw new UnauthorizedException('Invalid signature');
  }
  // ...
}
```

## Vérification

```bash
npx nx typecheck --all
```

## Risques

- **Moyen** : Le downgrade expo-file-system peut casser des APIs utilisées dans le mobile
- La vérification Meta nécessite `META_APP_SECRET` dans les variables d'environnement
