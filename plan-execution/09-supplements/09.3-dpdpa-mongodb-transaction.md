# 09.3 — dpdpa-consent : rendre la suppression MongoDB atomique avec PostgreSQL

## Contexte

`dpdpa-consent.service.ts` supprime les données MongoDB **hors transaction PostgreSQL**. Si MongoDB échoue, les données PostgreSQL sont supprimées mais pas MongoDB → **suppression partielle irréversible**.

L'action 05.4 corrige le scope (userId) mais pas l'atomicité.

## Action

Utiliser un pattern Saga ou Two-Phase Commit simplifié :

**Avant** (non atomique) :
```ts
await this.prisma.$transaction(async (tx) => {
  await tx.vital.deleteMany({ where: { patientId } });
  // ...
});
await this.mongoService.deletePatientData(patientId); // Hors transaction !
```

**Après** (saga avec compensation) :
```ts
try {
  // Phase 1 : Supprimer MongoDB d'abord
  await this.mongoService.deletePatientData(patientId);

  // Phase 2 : Supprimer PostgreSQL dans une transaction
  await this.prisma.$transaction(async (tx) => {
    await tx.vital.deleteMany({ where: { patientId, recordedBy: userId } });
    // ...
  });
} catch (error) {
  // Compensation : restaurer MongoDB si PostgreSQL échoue
  // (nécessite une sauvegarde préalable ou un undo log)
  this.logger.error('Suppression partielle — compensation nécessaire', error);
}
```
