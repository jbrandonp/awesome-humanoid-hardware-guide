# 09.1 — Corriger tauri.conf.json : frontendDist, CSP, identifier, capabilities

**Réf** : `260426.md:297` (C13) + `260426.md:311` (C22) + `260426.md:312` (C23) + `260426.md:740-744` (H.3)  
**Priorité** : 🟠 MANQUANT — BLOQUANT DESKTOP BUILD + SÉCURITÉ  
**Durée estimée** : 10 min  
**Dépend de** : P1

---

## Contexte

4 bugs dans la config Tauri :
1. **C13** : `frontendDist` pointe vers `../dist/apps/desktop` mais Vite output vers `apps/desktop/dist` → build desktop cassé
2. **C22** : `"csp": null` → XSS possible dans le webview
3. **C23** : `"identifier": "com.tauri.dev"` → placeholder par défaut
4. **H.3** : `capabilities/default.json` n'a pas les permissions pour le plugin `dialog` → bloqué

---

## Fichiers concernés

```
apps/desktop/src-tauri/tauri.conf.json
apps/desktop/src-tauri/capabilities/default.json
```

---

## Action

### 1. `tauri.conf.json`

**Avant** :
```json
{
  "identifier": "com.tauri.dev",
  "frontendDist": "../dist/apps/desktop",
  "app": {
    "security": {
      "csp": null
    }
  }
}
```

**Après** :
```json
{
  "identifier": "com.systeme-sante.desktop",
  "frontendDist": "../../apps/desktop/dist",
  "app": {
    "security": {
      "csp": "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'"
    }
  }
}
```

### 2. `capabilities/default.json`

Ajouter les permissions pour le plugin dialog :
```json
{
  "identifier": "main-capability",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "dialog:default"
  ]
}
```

---

## Vérification

```bash
cd apps/desktop
npx tauri build --debug
# Doit compiler sans erreur
```

---

## Risques

- **Moyen** : CSP peut bloquer des ressources légitimes → ajuster si nécessaire
- `identifier` doit être unique → utiliser un nom de domaine inversé réel
