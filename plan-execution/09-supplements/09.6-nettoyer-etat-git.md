# 09.6 — Nettoyer l'état Git : nbproject, 1455 unstaged files

**Réf** : `260426.md:758-760` (H.5) + `260426.md:776-779` (H.8)  
**Priorité** : 🟡 MANQUANT — GIT  
**Durée estimée** : 10 min  
**Dépend de** : P6

---

## Contexte

- **H.5** : `nbproject/` est un artefact IDE NetBeans avec potentiellement des chemins absolus locaux dans `private/private.properties`
- **H.8** : 1455 fichiers modifiés non stagés — probablement conversion CRLF→LF ou `prettier --write` global

---

## Fichiers concernés

```
nbproject/
.git/config (core.autocrlf)
```

---

## Action

### 1. Supprimer nbproject/ du tracking Git

```bash
git rm -r --cached nbproject/
echo "nbproject/" >> .gitignore
rm -rf nbproject/
```

### 2. Analyser les 1455 fichiers modifiés

```bash
git diff --stat | head -30
# Identifier si c'est CRLF→LF ou prettier
```

Si conversion CRLF :
```bash
git config core.autocrlf input
git add --renormalize .
git commit -m "chore: normalize line endings to LF"
```

Si prettier global :
```bash
git add -A
git commit -m "chore: apply consistent formatting"
```

---

## Vérification

```bash
git status
# Doit être clean (ou seulement les fichiers de P0-P8 modifiés)
```

---

## Risques

- **Élevé** : Commiter 1455 fichiers en masse peut masquer des changements réels
- **Recommandé** : Faire ceci en premier (avant P1) pour avoir un état Git clean
