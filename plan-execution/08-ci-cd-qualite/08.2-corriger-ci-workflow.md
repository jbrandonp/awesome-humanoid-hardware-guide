# 08.2 — Remplacer les placeholders dans ci.yml

**Réf** : `260426.md:1215-1217`  
**Priorité** : 🟢 P8 — CI/CD  
**Durée estimée** : 15 min  
**Dépend de** : P3

---

## Contexte

Le workflow CI contient des `echo` placeholders au lieu de vraies commandes de déploiement :

```yaml
- name: Deploy preview
  run: echo "Deploy preview"
- name: Deploy production
  run: echo "Deploy production"
```

---

## Fichier concerné

```
.github/workflows/ci.yml
```

---

## Action

### Preview deploy (Vercel recommandé)

```yaml
- name: Deploy preview
  run: |
    npx vercel --token ${{ secrets.VERCEL_TOKEN }} \
      --env DATABASE_URL=${{ secrets.PREVIEW_DATABASE_URL }} \
      --confirm
```

### Production deploy (kubectl)

```yaml
- name: Deploy production
  run: |
    kubectl set image deployment/api api=${{ secrets.DOCKERHUB_USERNAME }}/api:${{ github.sha }}
    kubectl rollout status deployment/api
```

---

## Vérification

Le workflow CI doit passer jusqu'au bout sans erreur.

---

## Risques

- **Moyen** : Nécessite de configurer Vercel/kubectl et les secrets associés
- Si aucun déploiement n'est prévu, laisser les `echo` mais ajouter un commentaire TODO
