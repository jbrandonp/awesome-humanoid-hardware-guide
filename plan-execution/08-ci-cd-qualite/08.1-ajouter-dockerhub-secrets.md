# 08.1 — Ajouter DOCKERHUB_USERNAME et DOCKERHUB_TOKEN aux secrets GitHub

**Réf** : `260426.md:1213`  
**Priorité** : 🟢 P8 — CI/CD  
**Durée estimée** : 5 min  
**Dépend de** : P3

---

## Contexte

Le workflow CI fait référence à `DOCKERHUB_USERNAME` et `DOCKERHUB_TOKEN` mais ces secrets ne sont probablement pas configurés dans le repo GitHub.

---

## Action

1. Aller dans **GitHub Repo → Settings → Secrets and variables → Actions**
2. Ajouter :
   - **Nom** : `DOCKERHUB_USERNAME` — **Valeur** : le username Docker Hub
   - **Nom** : `DOCKERHUB_TOKEN` — **Valeur** : un token d'accès Docker Hub (pas le mot de passe)

---

## Vérification

```bash
gh secret list
# Doit lister DOCKERHUB_USERNAME et DOCKERHUB_TOKEN
```

---

## Risques

- **Faible** : Configuration standard GitHub
- Le token Docker Hub doit avoir les permissions minimales (Read & Write packages)
