# 06.5 — Ajouter SmartOnFhirGuard et RolesGuard dans leurs modules

**Réf** : `260426.md:713-727` (services orphelins) + `260426.md:870-882` + `260426.md:1153-1154`  
**Priorité** : 🟡 P6 — CORRECTION DI  
**Durée estimée** : 5 min  
**Dépend de** : P1

---

## Contexte

`SmartOnFhirGuard` est utilisé dans `fhir.controller.ts` via `@UseGuards()` mais n'est pas dans les `providers` de `FhirModule`. `RolesGuard` est utilisé dans plusieurs contrôleurs mais absent de `AuthModule.providers`. NestJS peut parfois les instancier via `Reflector` global, mais ce n'est pas garanti.

---

## Fichiers concernés

```
apps/api/src/app/fhir/fhir.module.ts
apps/api/src/app/auth/auth.module.ts
```

---

## Action

### 1. `fhir.module.ts`

Ajouter dans `providers` :
```ts
providers: [/* ... */, SmartOnFhirGuard],
```

### 2. `auth.module.ts`

Ajouter dans `providers` :
```ts
providers: [/* ... */, RolesGuard],
```

---

## Vérification

```bash
npx nx typecheck api
```

---

## Risques

- **Faible** : Ajout de providers standard
- `RolesGuard` a besoin de `Reflector` → vérifier qu'il est bien injecté
