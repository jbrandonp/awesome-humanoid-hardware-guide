# 03.4 — Corriger Dockerfile.api CMD : chemin de sortie webpack

**Réf** : `260426.md:292` (C8) + `260426.md:827-833` + `260426.md:1034-1037`  
**Priorité** : 🔴 P3 — BLOQUANT DOCKER RUN  
**Durée estimée** : 3 min  
**Dépend de** : P1

---

## Contexte

Le `CMD` pointe vers `dist/apps/api/main.js`, mais webpack output vers `apps/api/dist/` (config webpack ligne 10). Le chemin correct est `apps/api/dist/main.js`.

---

## Fichier concerné

```
infra/docker/Dockerfile.api:35
```

---

## Action

**Avant** :
```dockerfile
CMD ["node", "dist/apps/api/main.js"]
```

**Après** :
```dockerfile
CMD ["node", "apps/api/dist/main.js"]
```

---

## Vérification

```bash
# Vérifier la config webpack
grep -A10 "output" apps/api/webpack.config.js | grep path
```

---

## Risques

- **Faible** : Correction de chemin
- Vérifier que le WORKDIR du Dockerfile est bien `/app` (pour que `apps/api/dist/main.js` soit relatif à `/app`)
