# 05.8 — Créer Patient système pour éviter FK violation

**Réf** : `260426.md:835-842` + `260426.md:1113-1118`  
**Priorité** : 🟠 P5 — INTÉGRITÉ DONNÉES  
**Durée estimée** : 10 min  
**Dépend de** : P1

---

## Contexte

`pacs-indexer.processor.ts` crée des `notificationStatus` avec `patientId: '00000000-0000-0000-0000-000000000000'`, mais ce patient n'existe pas en base → FK violation. Il faut un patient système.

---

## Fichiers concernés

1. `apps/api/src/app/prisma/prisma.service.ts`
2. `apps/api/src/app/pacs/pacs-indexer.processor.ts:192`

---

## Action

### 1. Dans `prisma.service.ts` → `onModuleInit()`

```ts
async onModuleInit() {
  await this.$connect();
  
  // Patient système
  await this.patient.upsert({
    where: { id: '00000000-0000-0000-0000-000000000000' },
    create: {
      id: '00000000-0000-0000-0000-000000000000',
      firstName: 'SYSTEM',
      lastName: 'SYSTEM',
      dateOfBirth: new Date('1900-01-01'),
    },
    update: {},
  });
}
```

### 2. Dans `pacs-indexer.processor.ts`

Vérifier que le `patientId` utilisé matche bien le patient système.

---

## Vérification

```bash
npx nx typecheck api
```

---

## Risques

- **Faible** : Ajout d'un seed, pas de migration
- Le patient système ne doit pas apparaître dans les listes de patients normales
