# 05.7 — Corriger generate-secrets.ps1 : mots de passe distincts

**Réf** : `260426.md:296` (C12) + `260426.md:643` (INF3-INF4) + `260426.md:1108-1110`  
**Priorité** : 🟠 P5 — SÉCURITÉ  
**Durée estimée** : 10 min  
**Dépend de** : P1

---

## Contexte

`generate-secrets.ps1` utilise `-replace` global sur TOUS les `CHANGE_ME_SECURE_PASSWORD`, ce qui met le même mot de passe partout (postgres, mongo, minio, JWT, etc.). Chaque service doit avoir un secret distinct.

---

## Fichier concerné

```
infra/kubernetes/generate-secrets.ps1:46-58
```

---

## Action

Utiliser des placeholders distincts dans le template ET générer des secrets distincts :

**Avant** :
```powershell
$allSecrets = (Get-Content $template) -replace 'CHANGE_ME_SECURE_PASSWORD', $password1
```

**Après** :
```powershell
$allSecrets = (Get-Content $template) `
  -replace 'POSTGRES_PASSWORD_PLACEHOLDER', (New-RandomPassword) `
  -replace 'MONGO_PASSWORD_PLACEHOLDER', (New-RandomPassword) `
  -replace 'MINIO_PASSWORD_PLACEHOLDER', (New-RandomPassword) `
  -replace 'JWT_SECRET_PLACEHOLDER', (New-RandomPassword) `
  -replace 'TOKEN_ENCRYPTION_KEY_PLACEHOLDER', (New-RandomPassword)
```

---

## Vérification

```bash
# Générer les secrets et vérifier qu'ils sont tous différents
pwsh infra/kubernetes/generate-secrets.ps1
grep -oP 'password:\s*\K.*' infra/kubernetes/secrets.yaml | sort | uniq -c
# Chaque mot de passe doit apparaître 1 fois
```

---

## Risques

- **Moyen** : Nécessite de mettre à jour le template YAML avec les nouveaux placeholders
- Les références aux anciens placeholders doivent être mises à jour dans tout le code
