# 05.5 — Ajouter colonne glucose dans le schéma WatermelonDB + Zod

**Réf** : `260426.md:659` (LB1-LB3-LB5) + `260426.md:1098-1101`  
**Priorité** : 🟠 P5 — INTÉGRITÉ DONNÉES  
**Durée estimée** : 10 min  
**Dépend de** : P1

---

## Contexte

La colonne `glucose` est déclarée dans le modèle TypeScript (`Vital.glucose`) mais **absente** du schéma WatermelonDB ET du schéma Zod. Les données glycémiques (BLE, OCR) sont perdues silencieusement.

---

## Fichiers concernés

1. `libs/models/src/lib/schema.ts` (WatermelonDB)
2. `libs/models/src/lib/models.ts` (Zod)

---

## Action

### 1. WatermelonDB (`schema.ts`) — dans la table `vitals`

Ajouter :
```ts
{ name: 'glucose', type: 'number', isOptional: true },
```

### 2. Zod (`models.ts`) — dans `VitalsSchema`

Ajouter :
```ts
glucose: z.number().optional().nullable(),
```

---

## Vérification

```bash
npx nx typecheck models
npx nx test models
```

---

## Risques

- **Faible** : Ajout de colonne, pas de breaking change
- Les données existantes auront `glucose: null`
