# 07.4 — Corriger le split HL7 (\\n → \\r)

**Réf** : `260426.md:668` (LB10) + `260426.md:1179-1180`  
**Priorité** : 🟡 P7 — FONCTIONNEL  
**Durée estimée** : 3 min  
**Dépend de** : P1

---

## Contexte

Le protocole HL7 utilise `\r` (CR, caractère 13) comme séparateur de segments, pas `\n` (LF). Le split sur `\n` échoue sur de vrais messages HL7 → parsing incorrect.

---

## Fichier concerné

```
apps/api/src/app/hl7/hl7-mllp.controller.ts:171
```

---

## Action

**Avant** :
```ts
const segments = message.split('\n');
```

**Après** :
```ts
const segments = message.split(/\r?\n/) || message.split('\r');
```

Ou mieux :
```ts
const segments = message.split('\r').map(s => s.trim()).filter(Boolean);
```

## Vérification

---

## Risques

- **Faible** : Changement de caractère de split
- Certains systèmes envoient `\r\n` → la regex `\r?\n` gère les deux cas
