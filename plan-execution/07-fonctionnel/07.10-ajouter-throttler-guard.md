# 07.10 — Ajouter ThrottlerGuard dans app.module.ts

**Réf** : `260426.md:1204-1205`  
**Priorité** : 🟡 P7 — SÉCURITÉ  
**Durée estimée** : 5 min  
**Dépend de** : P1

---

## Contexte

Le `ThrottlerGuard` (rate limiting) n'est pas enregistré globalement. L'API n'a **aucune protection** contre le brute-force, DoS, ou scraping.

---

## Fichier concerné

```
apps/api/src/app/app.module.ts
```

---

## Action

Ajouter dans le tableau `providers` :

```ts
providers: [
  // ... existants
  { provide: APP_GUARD, useClass: ThrottlerGuard },
],
```

Vérifier que `ThrottlerModule` est bien importé dans `imports` avec une config :

```ts
imports: [
  ThrottlerModule.forRoot([{
    ttl: 60000,
    limit: 10,
  }]),
  // ...
],
```

---

## Vérification

```bash
# Envoyer 11 requêtes en <60s → la 11ème doit retourner 429
for i in $(seq 1 11); do
  curl -s -o /dev/null -w "%{http_code}\n" http://localhost:3000/api/auth/login
done
# Les 10 premières → 4xx (hors rate limit)
# La 11ème → 429 Too Many Requests
```

---

## Risques

- **Faible** : Ajout standard NestJS
- Ajuster `ttl` et `limit` selon les besoins de production
