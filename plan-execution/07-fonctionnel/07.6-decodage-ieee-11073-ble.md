# 07.6 — Implémenter le décodage IEEE 11073 SFLOAT pour le BLE

**Réf** : `260426.md:164` (F2) + `260426.md:395` (BLE1-BLE2) + `260426.md:1186-1190`  
**Priorité** : 🟡 P7 — CRITIQUE CLINIQUE (données vitales erronées)  
**Durée estimée** : 20 min  
**Dépend de** : P1

---

## Contexte

Les données BLE (pression artérielle, glucose) sont lues comme des **UInt16 bruts** au lieu d'être décodées en **IEEE 11073 SFLOAT** (mantissa × 10^exponent). Résultat : **données vitales 100% fausses**.

## Action

### 1. Fonction de décodage SFLOAT IEEE 11073

```ts
function decodeSFLOAT(rawValue: number): number {
  const mantissa = rawValue & 0x0FFF;
  const exponent = (rawValue >> 12) & 0x0F;
  // Gérer le signe (twos complement sur 12 bits)
  const signedMantissa = mantissa >= 0x0800 ? mantissa - 0x1000 : mantissa;
  return signedMantissa * Math.pow(10, exponent);
}
```

### 2. Pression artérielle (offset correct)

```ts
// offset 1 = systolic (SFLOAT), offset 3 = diastolic (SFLOAT), offset 5 = mean arterial pressure (SFLOAT)
const systolic = decodeSFLOAT(data.readUInt16LE(1));
const diastolic = decodeSFLOAT(data.readUInt16LE(3));
```

### 3. Glucose (offset correct)

```ts
// offset 1 = sequence number (UInt16, ignorer)
// offset 3 = date/time (7 bytes)
// offset 10 = glucose concentration (SFLOAT)
const glucose = decodeSFLOAT(data.readUInt16LE(10));
```

## Vérification

## Risques

- **Élevé** : Impact clinique direct — les données affichées aux médecins changent
- Vérifier avec un vrai dispositif BLE médical (tensiomètre)
- Les offsets peuvent varier selon le fabricant → prévoir une table de mapping
