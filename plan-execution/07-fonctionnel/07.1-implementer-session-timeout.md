## Contexte

`SessionTimeoutMiddleware` est un **no-op**. Il appelle `next()` sans aucune logique. Aucun timeout de session n'est appliqué, malgré le paramètre `SESSION_TIMEOUT_MINUTES=30` dans `.env.example`.

## Action

**Avant** (vide) :
```ts
@Injectable()
export class SessionTimeoutMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: NextFunction) {
    next();
  }
}
```

**Après** (implémenté) :
```ts
@Injectable()
export class SessionTimeoutMiddleware implements NestMiddleware {
  constructor(private jwtService: JwtService) {}

  use(req: Request, res: Response, next: NextFunction) {
    const token = req.headers.authorization?.replace('Bearer ', '');
    if (!token) return next();

    try {
      const payload = this.jwtService.verify(token);
      const now = Math.floor(Date.now() / 1000);
      const timeout = parseInt(process.env.SESSION_TIMEOUT_MINUTES || '30') * 60;

      if (payload.iat && (now - payload.iat) > timeout) {
        throw new UnauthorizedException('Session expirée');
      }
    } catch (e) {
      if (e instanceof UnauthorizedException) throw e;
      // Token invalide → laisser AuthGuard gérer
    }

    next();
  }
}
```

## Vérification

```bash
# Tester avec un token expiré (iat > 30 min)
curl -H "Authorization: Bearer <token_expire>" http://localhost:3000/api/patient
# Doit retourner 401 "Session expirée"
```
