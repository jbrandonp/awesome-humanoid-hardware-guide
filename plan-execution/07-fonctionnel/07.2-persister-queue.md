## Contexte

La file d'attente des patients est stockée dans un simple `Array` en mémoire. Au redémarrage du serveur, toutes les files d'attente sont **perdues**. Il faut persister dans Redis (préféré) ou PostgreSQL.

## Action

### Option A — Redis (recommandé)

```ts
@Injectable()
export class QueueService {
  private readonly QUEUE_KEY = 'patient:queue';

  constructor(private redis: RedisService) {}

  async addPatientToQueue(patientId: string, priority: number) {
    await this.redis.zadd(this.QUEUE_KEY, priority.toString(), patientId);
  }

  async getNextPatient(): Promise<string | null> {
    const result = await this.redis.zpopmin(this.QUEUE_KEY);
    return result?.[0]?.element ?? null;
  }

  async getQueueLength(): Promise<number> {
    return this.redis.zcard(this.QUEUE_KEY);
  }
}
```

### Option B — PostgreSQL (fallback)

Utiliser une table `QueueEntry` via Prisma (moins performant pour du temps réel).
