# 07.7 — Remplacer Buffer par API React Native

**Réf** : `260426.md:623` (FR1) + `260426.md:846-850` + `260426.md:1192-1195`  
**Priorité** : 🟡 P7 — FONCTIONNEL  
**Durée estimée** : 10 min  
**Dépend de** : P1

---

## Contexte

`background-sync.service.ts` utilise `Buffer.from()` qui est une API Node.js **non disponible en React Native**. Le code crash au runtime sur mobile.

---

## Fichier concerné

```
apps/mobile/src/services/background-sync.service.ts:81
```

---

## Action

### Option A — Utiliser `atob()` natif + `TextDecoder`

**Avant** :
```ts
const decoded = Buffer.from(payloadBase64, 'base64').toString('utf-8');
```

**Après** :
```ts
const decoded = decodeURIComponent(
  Array.from(atob(payloadBase64))
    .map(c => '%' + ('00' + c.charCodeAt(0).toString(16)).slice(-2))
    .join('')
);
```

### Option B — Installer le package `buffer`

```bash
npm install buffer
```

```ts
import { Buffer } from 'buffer';
const decoded = Buffer.from(payloadBase64, 'base64').toString('utf-8');
```

## Risques

- **Faible** : Le package `buffer` est déjà une dépendance transitive (via WatermelonDB)
- L'option A (atob) est plus légère mais moins lisible
